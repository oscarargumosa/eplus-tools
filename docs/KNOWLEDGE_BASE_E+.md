# Base de Conocimiento E+ — Catálogo de fuentes

> Trabajo interno de investigación y recopilación (2026-06-23). Objetivo: mapear **qué contenido
> existe** sobre cómo escribir proyectos Erasmus+, decidir cuál **copiamos / mejoramos / creamos /
> descartamos**, y a partir de ahí producir el material de la academia.
>
> **NO es contenido público todavía.** Es el insumo para decidir el plan editorial.
> Este `.md` es la versión revisable; una vez Óscar marque estados, se normaliza a
> `data/knowledge_base/sources.json` y el VPS monta el visor (ver `docs/handoffs/PARA_VPS.md`).

## Cómo leer este documento

- **Tronco común (A–J):** conocimiento transversal, vale para todas las especialidades.
- **Especialidades (1–9):** un curso por temática. *Sin universidades / Educación Superior / Jean Monnet.*
- **Estado sugerido** = recomendación provisional del barrido. **Óscar decide el estado final**
  sobrescribiendo esta columna:
  - `copiar` = fuente buena y replicable casi tal cual / referencia base.
  - `mejorar` = hay base útil pero hay que adaptarla, actualizarla o reescribirla mejor.
  - `crear` = no existe buen material → contenido propio (potencial activo diferenciador).
  - `descartar` = no aporta o es solo herramienta/landing para enlazar, no para replicar.
- **Calidad** 1–5 (5 = imprescindible / fuente madre).

## 6. Síntesis — Lo que NO existe y deberíamos CREAR (contenido premium / activo diferenciador)

El barrido confirma un patrón claro: **el material oficial UE/AN cubre muy bien elegibilidad,
criterios y presupuesto, pero NO enseña a *redactar*.** Ese es precisamente nuestro hueco de valor,
y encaja con activos que ya tenemos internos (Diagnose, pattern library de evaluadores, leyes EACEA).

**Huecos transversales (alta prioridad — `crear`):**
1. **Teoría del cambio / lógica de intervención (C):** la Comisión no publica método; solo lo pide.
2. **Objetivos SMART e indicadores medibles (F):** se exigen pero no hay guía oficial de cómo formularlos.
3. **Lenguaje de evaluador / evidencia vs afirmación (F):** conocimiento tácito, sin fuente oficial. Premium.
4. **Catálogo de errores y ejemplos de rechazo reales (J):** ninguna AN lo publica pese a tener los datos. Nuestro pattern library de cartas de evaluador es justo este activo.
5. **Todo lo anterior EN ESPAÑOL:** las pocas guías de técnica buenas están en inglés o detrás de webinars no descargables.

**Huecos por especialidad:**
- **KA1 Juventud / Adultos:** falta guía en español de *cómo redactar* (SALTO está en EN; INJUVE es descriptivo).
- **KA1 VET:** poco material sobre **ErasmusPro** (movilidades largas 90-365 días) y rol de orgs de apoyo.
- **KA1 Escolar / Adultos:** falta guía de **presupuesto** y de **criterios (E)** en español.
- **KA220 / KA210:** no hay rúbrica pública con descriptores finos por criterio ni ejemplos de cómo justificar el tramo lump sum (30k vs 60k). El "Handbook KA2 lump sums" oficial está tras EU Login.
- **KA3:** sin guía de redacción en **español** para acciones EACEA; sin desglose Part B↔criterios.
- **Capacity Building (8):** sin material divulgativo en español (SEPIE/INJUVE no gestionan EACEA centralizado); más allá de los "Tips for applicants" no hay texto modelo de Part B ganador.
- **Deporte (9):** sin fuente española oficial verificable; estructura del Part B / criterios poco expuestos.

**Señales de demanda (qué busca el público, repetido en casi todas las fuentes):**
- "Escribe para el evaluador": trocear preguntas, bullets, palabras clave, valor añadido explícito.
- Cadena lógica necesidades → objetivos SMART → actividades → resultados → impacto.
- Los **work packages** justifican el lump-sum (no hay desglose financiero): WP débiles = puntuación baja.
- El **impacto** es lo peor desarrollado y donde se ganan/pierden puntos.
- Coherencia narrativa↔presupuesto + revisión externa antes de enviar.
